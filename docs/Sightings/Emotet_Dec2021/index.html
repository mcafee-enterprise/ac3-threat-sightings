Emotet_Dec2021

Overview#

This Threat Sighting documents observed Emotet infection followed by Cobalt Strike activity. Weaponized .xlsm delivered via Email. User execution via Excel. Execution and C2 via RunDll32. Process injection. Masqueraded CERTUTIL. Cobalt Strike stagers. Discovery and Persistence. Victim had protection solution configured to do-not-block.

This document is a summary of the extracted TTPs. For full details please refer to the AC3 Threat Sighting (.yml file) at: https://github.com/mcafee-enterprise/ac3-threat-sightings/blob/main/sightings/Sightings_Emotet_Dec2021.yml

Author: Alejandro Houspanossian (@lekz86)

Acknowledgements: Alejandro Houspanossian (@lekz86)

Threat Actor Activity#

  • Threat Actor used Email with .XLSM file in attachment for Initial Access.
  • Threat actor used weaponized Microsoft Office files for Execution.
  • Threat Actor used Embedded Office Macro to download .dll files from external servers for Command And Control.
  • Threat Actor used Embedded Office Macro to execute .dll files via RunDll32 for Defense Evasion.
  • Threat actor used nested invocations of RunDll32 for Defense Evasion.
  • Threat actor used HTTP connections to downloaded a masqueraded copy of CERTUTIL for Command And Control.
  • Threat actor used Windows Registry Autorun keys for Persistence.
  • Threat actor used RunDll32 to launch and inject CERTUTIL for Execution.
  • Threat actor used CERTUTIL to access Web Browser credentials stores for Credential Access.
  • Threat actor used HTTP to download Cobalt Strike stager DLL for Command And Control.
  • Threat actor used RegSvr32 to execute Cobalt Strike Stager DLL for Execution.
  • Threat actor used process injection into mstsc.exe to launch Cobalt Strike post-exploitation tasks for Command And Control.
  • Threat actor used Cobalt Strike to launch system native tools for Discovery.
  • Threat actor used HTTP to downloaded additional Cobalt Strike dll files for Command And Control.
  • Threat actor used SCHTASK to create an scheduled task that executes the Cobalt Strike beacon for Persistence.
  • Threat actor used RunDll32 via an Scheduled Task to execute Cobalt Strike Stager DLL for Command And Control.
  • Threat actor used Cobalt Strike to perform hands-on-keyboard discovery for Discovery.
  • Threat actor used Reg.exe to attempt a known Elevation Control Mechanism bypass for Privilege Escalation.
  • Threat actor used Cobalt Strike for initiating command execution on remote hosts via CMD and WMIC for Discovery.
  • Threat actor used WMIC via Cobalt Strike to list existing Volume Shadow Copies via vssadmin.exe for Discovery.
  • Threat actor used HTTP connections for Command And Control.
