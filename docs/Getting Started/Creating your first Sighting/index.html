Creating your first sighting

This article describes how to create your first Threat Sighting step by step.

BEFORE YOU BEGIN

It's important before you begin with your first Threat Sighting creation that you have read the schema overview section. If you didn't, please, go and understand each field available in the current version.

Standard Sections#

As you will know, there are required sections in Threat Sightings. Therefore, when you are ready to create a Threat Sighting, you need to have clear which are those sections in order to add it.

All the Threat Sightings must to have the following sections.

  1. header
  2. threatSightings
  3. behavior
  4. footer

Let's start with the required fields#

There isn't order to create a Threat Sighting. However, based in our experience, is interesting follow the order defined in this section in the time when you are decided to create a Threat Sighting.

Step 1 - Create your template or start from scratch#

When you are going to create a Threat Sighting, you will have a template with the required fields introduced or you can start from scratch writing the YAML file. We recommend use a template, since it save you time.

The following template can be a good start.

---header:  sightingReportId:   status:   description:   author:   acknowledgement:   tlp:   threatInformation:    adversaries:    -     malware:    -     tools:    -     lolbas:    -     regions:    -     industries:    - threatSightings:  - sighting:     id:     behaviors:       - behavior:       id:       type:       weapon:footer:  changeTracking:    created:     lastModified:     sightingVersion:    schemaVersion:   references:    -
IMPORTANT

Note that some fields will change in future schema versions. You can check the latest version in our changelog.

https://github.com/mcafee-enterprise/ac3-threat-sightings/blob/main/CHANGELOG.md

Step 2 - Fill the header section#

In this step you will start filling the Threat Sighting. As we said, there isn't order, but start for the beginning always is important! Let's imagine that we are creating a Threat Sighting related to TA551. In the campaign or intrusion that we want cover they have used Valak malware and differents tools to achieve their objectives.

First, fill the information related to description, author, id, etc...

---header:  sightingReportId: 62cbd3e0-e672-4251-b3cc-774267b693fc  status: wip  description: TA551 (Shatak) is a financially motivated cybercrime group. In this campaign, they have used Valak and different LOLBAS to establish C2 communications and exfiltrate corporate information.  author: AC3 Threat Sightings - Trellix  acknowledgement: Trellix  tlp: white

Remember follow the documentation for each field in the Schema Overview section.

With this information completed, from our point of view it is more interesting to fill the threatSighting information than the threatInformation, because while you are adding behaviors, you will discover new tools or lolbas used in the campaign. So, finally will be a good practice check all the behaviors in the end and get all the tools used for put it in the threatInformation section.

Step 3 - Now focus on behaviors#

The Step 3 is focused in three elements mainly, threatSightings, sighting and behavior sections. If you remember, each YAML file has only one threatSighting section, however, each threatSighting section can have different sighting sections in turn. And finally, each sighting can have many behaviors as you want to add.


graph LR; threatSighting-->Sighting_1; threatSighting-->Sighting_2; threatSighting-->Sighting_3; Sighting_1-->behavior_1_1; Sighting_1-->behavior_1_2; Sighting_2-->behavior_2_1; Sighting_2-->behavior_2_2; Sighting_2-->behavior_2_3; Sighting_3-->behavior_3_1;

In this example, let's suppose that the intrusion that we are analyzing has two sightings, one related to execution tactic and another one related to C2 tactic. And, the execution tactic has two behaviors while that C2 tactic has only one. Therefore, we are going to add those sightings.

Step 4 - Finish with the footer#

In order to finish with these required fields, let's fill the footer section with the tracking information of your Threat Sighting. The information into this section is based purely on tracking information about the dates and versions. Also, if you have URLs where you got information, is recommended add it into the references field.

Step 5 - Remember the threatInformation!!#

At this point, do you remember when we said in the step 2 that we recommend fill first the threatSighting section than threatInformation section? Then, don't forget to complete it now!

In the header section we need add the threatInformation section with all information about the malware, lolbas, tools, etc... used by the actor, in this case, TA551.

Let's imagine that TA551 has used the following components.

Malware

  • Valak
  • Trickbot

Tools

  • Empire
  • Cobalt Strike

LOLBAS

  • regsvr32.exe
  • mshta.exe
  • winword.exe
  • cmd.exe

Affected regions

  • Western Europe:
    • Germany
  • Northern America:
    • United States of America

Affected sectors

  • Financial

It's time to add this information into our Threat Sighting
